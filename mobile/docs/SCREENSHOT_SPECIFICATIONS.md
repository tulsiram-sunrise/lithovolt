## 📱 Technical Specifications

### Android (Google Play Store)

**Phone Screenshots**:
```
Format: PNG or JPEG
Minimum dimensions: 320px (shortest side)
Maximum dimensions: 3840px (shortest side)
Aspect ratio: 16:9 recommended
Recommended size: 1080 x 1920 (portrait) or 1920 x 1080 (landscape)
Quantity: Minimum 2, Maximum 8
File size: Under 8MB each
```

**Tablet Screenshots** (Optional):
```
Format: PNG or JPEG
Minimum dimensions: 1200 x 1920
Maximum dimensions: 7680px (shortest side)
Quantity: Maximum 8
Recommended: 7" and 10" tablet sizes
```

### iOS (App Store)

**Required Sizes**:
```
6.5" Display (iPhone 14 Pro Max, 13 Pro Max, 12 Pro Max, 11 Pro Max):
- Size: 1284 x 2778 (portrait) or 2778 x 1284 (landscape)

5.5" Display (iPhone 8 Plus, 7 Plus, 6s Plus):
- Size: 1242 x 2208 (portrait) or 2208 x 1242 (landscape)

iPad Pro (3rd gen) 12.9":
- Size: 2048 x 2732 (portrait) or 2732 x 2048 (landscape)

iPad Pro (2nd gen) 12.9":
- Size: 2048 x 2732 (portrait) or 2732 x 2048 (landscape)
```

**Guidelines**:
- Minimum 2 screenshots per device size
- Maximum 10 screenshots per device size
- File format: PNG or JPEG (PNG recommended)
- Color space: RGB
- No transparency
- 72 DPI minimum

## 🎨 Design Guidelines

### Brand Elements

**Colors**:
- Primary: `#667eea` (Purple)
- Secondary: `#764ba2` (Dark Purple)
- Success: `#4caf50` (Green)
- Background: `#ffffff` (White)
- Text: `#333333` (Dark Gray)
- Accent: Gradients using primary and secondary

**Typography**:
- Headlines: Bold, sans-serif (Segoe UI, Roboto, SF Pro)
- Body: Regular weight, high readability
- Size: Large enough to read at thumbnail size

**Spacing**:
- Generous whitespace
- Consistent margins (40-60px from edges)
- Visual hierarchy with size and color

### Mockup Style Options

#### Option 1: Device Frame
```
Pros:
✓ Realistic presentation
✓ Professional appearance
✓ Context for users

Cons:
✗ Reduces usable space
✗ Device may date quickly
✗ Distracts from content

Best for: Consumer-focused apps
```

#### Option 2: Floating UI (Recommended)
```
Pros:
✓ Maximum content visibility
✓ Timeless design
✓ Focus on features
✓ Easy to update

Cons:
✗ Less realistic
✗ Requires good design

Best for: B2B apps (Lithovolt)
```

#### Option 3: Hybrid
```
Pros:
✓ Balance of both approaches
✓ Flexibility

Cons:
✗ Can be cluttered
✗ Harder to execute well

Best for: Mixed B2B/B2C apps
```

**Recommendation for Lithovolt**: **Option 2 - Floating UI** with text overlays

## 🎬 Screenshot Creation Workflow

### Step 1: Capture Base Screenshots
1. Use actual app running on device
2. Populate with realistic data (not Lorem Ipsum)
3. Ensure UI is pixel-perfect
4. Capture at highest device resolution
5. Use consistent device orientation (portrait recommended)

**Tools**:
- Android: ADB screenshot command or device built-in
- iOS: Xcode Simulator or physical device
- Screen recording: For video demonstrations

### Step 2: Edit and Enhance
1. **Crop and Resize**: Match store requirements exactly
2. **Remove Status Bar** (optional): Clean look
3. **Add Background**: Brand gradient or solid color
4. **Apply Device Frame** (if using): Mockup templates

**Recommended Tools**:
- **Figma** (Free, web-based, collaborative)
- **Adobe Photoshop** (Professional, paid)
- **Sketch** (Mac only, paid)
- **Canva** (Easy templates, freemium)
- **Placeit by Envato** (Device mockup generator)

### Step 3: Add Text Overlays
1. **Headline**: Bold, 48-72pt, top of frame
2. **Subtext**: Regular, 24-36pt, below headline
3. **Colors**: High contrast (white on dark, dark on light)
4. **Position**: Consistent across all screenshots
5. **Alignment**: Center or left-aligned

**Typography Tips**:
- Use no more than 2 font weights
- Keep text to 1-2 lines maximum
- Ensure readability at thumbnail size (test at 200px wide)

### Step 4: Quality Check
- [ ] Resolution matches store requirements
- [ ] File size under limits
- [ ] Text is readable at thumbnail size
- [ ] Colors are consistent with brand
- [ ] No personal or sensitive information visible
- [ ] Proper file naming convention
- [ ] Screenshots tell a cohesive story
- [ ] Call-to-action is clear

### Step 5: Localization (Future)
For international markets:
- Translate text overlays
- Adapt cultural references
- Use region-appropriate imagery
- Follow local regulations

## 🎨 Background Design Options

### Option 1: Gradient Background
```css
background: linear-gradient(135deg, #667eea 0%, #764ba2 100%)
```
- Most aligned with brand
- Professional and modern
- High contrast for white screenshots

### Option 2: Solid Color
```css
background: #667eea (primary purple)
```
- Clean and simple
- Fast to produce
- Less dynamic

### Option 3: White Background
```css
background: #ffffff
```
- Minimal and clean
- Shows UI clearly
- Less branded feel

### Option 4: Abstract Pattern
- Subtle geometric patterns
- Battery/lightning motifs
- Not distracting from screenshot

**Recommendation**: **Option 1 - Gradient** for brand consistency

## 📝 Content Guidelines

### Do's ✅
- Use actual app screenshots (not mockups)
- Show realistic data (not placeholder text)
- Maintain consistent style across all screenshots
- Highlight unique features
- Use high-resolution images
- Test readability at small sizes
- Follow store guidelines strictly
- Update screenshots with major UI changes

### Don'ts ❌
- Don't include outdated UI
- Don't use competitor branding
- Don't show personal information
- Don't use Lorem Ipsum or fake data
- Don't make false claims
- Don't include prices (can change)
- Don't use low-resolution images
- Don't clutter with too much text
- Don't include device status bar with notifications

## 🎓 Best Practices from Industry Leaders

### Key Learnings
1. **First Screenshot Matters Most**: 60% of users only see the first screenshot
2. **Show Value Quickly**: Highlight key benefit in first frame
3. **Use Faces Sparingly**: Can distract from UI
4. **Consistency Wins**: Same style across all screenshots
5. **Caption Every Screenshot**: Reinforce message with text
6. **Mobile-First**: Most views are on mobile devices
7. **Localize Everything**: Translated screenshots perform 30% better
